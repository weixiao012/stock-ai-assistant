# 贡献规范

## 协作入口

开始开发前先阅读：

1. `AGENTS.md`
2. `README.md`
3. `docs/Codex多人开发规范.md`
4. `docs/MVP方案.md`

## 分支规则

- `main`：稳定分支
- `feature/<任务编号>-<模块名>`：新功能
- `fix/<任务编号>-<问题名>`：缺陷修复
- `docs/<任务编号>-<主题>`：文档任务

## 提交流程

1. 领取明确任务
2. 从最新 `main` 创建分支
3. 让 Codex 读取 `AGENTS.md` 和相关文档
4. 完成功能、自测和必要文档
5. 按 `.github/pull_request_template.md` 发起 PR
6. 评审通过后合并

## 评审重点

- 是否影响行情数据、预测逻辑、AI 分析或风险提示
- 是否有验证说明
- 是否保留金融风险边界
- 是否更新了相关文档
- 是否说明了 Codex 产出范围和人工复查结果
