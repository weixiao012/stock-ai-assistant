# MVP 方案

## 产品定位

A 股智能分析助手。

当前项目已有预测记录、前端页面、服务端接口和部署配置。后续应在现有基础上继续增强，而不是另起新仓库。

## 当前优先方向

| 编号 | 功能 | 说明 | 优先级 |
| --- | --- | --- | --- |
| M001 | 现有预测能力整理 | 梳理当前 `server.js`、`public/` 和 `data/` 的预测流程 | P0 |
| M002 | 个股智能分析 | 输入股票代码后生成结构化分析、风险点和追问建议 | P0 |
| M003 | 预测结果复盘 | 记录预测、实际走势、偏差原因和改进结论 | P0 |
| M004 | 数据源稳定性 | 明确 A 股数据来源、失败兜底和缓存策略 | P0 |
| M005 | 风险提示 | 所有预测和分析必须展示不保证收益说明 | P0 |
| M006 | 多人 Codex 开发流程 | 使用 `AGENTS.md`、PR 模板和规范文档统一协作 | P0 |

## 验收标准

- 保留现有项目功能，不因文档整合破坏运行
- README 清楚说明主仓库和项目边界
- 多人开发者能根据 `AGENTS.md` 开始任务
- PR 模板能统一 Codex 交接
- 不出现保证收益或确定性荐股表达
