# Codex 多人开发规范

## 协作模式

本项目采用“多人开发者 + GitHub + Codex”的协作模式。

每个开发者都可以使用自己的 Codex 开发功能，但必须围绕同一个主仓库 `weixiao012/stock-ai-assistant`、同一套文档规范和同一套 PR 规则工作。

## 每个开发者开工前

1. `git pull`
2. 查看 `AGENTS.md`
3. 查看 `docs/MVP方案.md`
4. 明确本次任务边界
5. 从 `main` 创建功能分支

## 推荐任务粒度

每个 Codex 任务尽量控制在一个清晰边界内：

- 一个页面
- 一个 API
- 一个数据模型
- 一个分析模块
- 一个测试集
- 一个文档主题

不要把“重做整个系统”作为一个 Codex 任务。

## Codex 交接格式

每次任务完成后，在 PR 或任务记录中留下：

```text
任务：
分支：
改动：
验证：
风险：
下一步：
```

## 冲突处理

- 先拉取最新 `main`
- 小冲突由当前任务负责人解决
- 涉及架构、数据模型、接口的大冲突，先写文档记录再改
- 不允许直接覆盖他人文件来省事
